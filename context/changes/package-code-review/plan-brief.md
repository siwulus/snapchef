# Independent Code-Review Package (Claude Code SDK) — Plan Brief

> Full plan: `context/changes/package-code-review/plan.md`

## What & Why

Add a fully independent package at `packages/code-review`: a terminal CLI that takes a **git diff on stdin** and produces a structured code review using the **Claude Code SDK** as its engine. The review is generated by forcing the model to call a single mandatory **`review` tool** whose input is a **Zod** schema — so the model's output _is_ a validated call to `review` (requirement #6), and structured I/O is Zod-validated end to end (requirement #4). Goal: an initial, locally-runnable reviewer — diff in, review out.

## Starting Point

The repo is a single Astro app using pnpm `11.6.0` (Node 24). `pnpm-workspace.yaml` exists but has no `packages:` glob and there is no `packages/` directory, so it is not yet a monorepo. The app reaches LLMs through `@openrouter/sdk`, not the Anthropic SDK, so the Claude Code SDK is new ground. `zod@^4` is already a dependency.

## Desired End State

After one root `pnpm install`, `git diff | pnpm --filter code-review review` prints a readable review (verdict + summary + findings); `--json` emits the raw validated object; `--model` overrides the model. `ANTHROPIC_API_KEY` is the only runtime credential. The Astro app's build and lint are untouched.

## Scope

**In scope:** the package skeleton + workspace wiring; Zod schemas (review output + CLI options); the Claude Code SDK engine with the mandatory `review` tool; the CLI (stdin → engine → rendered output) with `--json`/`--model`; minimal vitest unit tests.

**Out of scope:** CI/GitHub Action, PR-comment posting, git-hook integration; the tool running `git` itself; multi-file fan-out, diff chunking, retry/repair, caching; native `outputFormat` or JSON-parsing paths; a compiled/published binary; adopting the app's Effect/hexagonal conventions; deriving exit code from `verdict`.

## Architecture / Approach

`cli.ts` reads the diff from stdin and parses flags (`node:util` `parseArgs` → `CliOptions` Zod schema) → `engine.runReview(diff, { model })` defines a `review` tool from the `Review` Zod shape via `tool()` + `createSdkMcpServer()`, runs a single-shot headless `query()` (`tools: []`, `allowedTools: ["mcp__code-review__review"]`, `permissionMode: "bypassPermissions"`, `maxTurns: 1`), and captures the SDK-validated args in the tool handler → `render.ts` prints pretty text or JSON. Runtime deps: `@anthropic-ai/claude-agent-sdk` + `zod`.

## Phases at a Glance

| Phase                         | What it delivers                                                   | Key risk                                                                                           |
| ----------------------------- | ------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------- | --------------------------------------------------- |
| 1. Scaffold + workspace       | Package installs as a workspace member; root build/lint unaffected | Must exclude `packages/**` from the app's type-checked ESLint; SDK may need an `allowBuilds` entry |
| 2. Zod contracts + prompt     | `Review` + `CliOptions` schemas and the review prompt              | Schema shape must serve both the `tool()` input and the renderer                                   |
| 3. SDK engine (`review` tool) | `runReview(diff, {model}) → Review` via forced tool call           | SDK API names are version-sensitive; "model never called the tool" must be handled                 |
| 4. CLI + rendering            | End-to-end `git diff                                               | … review`, pretty + `--json`, error paths                                                          | stdin reading, clear errors (no diff / missing key) |

**Prerequisites:** an `ANTHROPIC_API_KEY` for manual end-to-end verification; pnpm 11 / Node 24 (already in place).
**Estimated effort:** ~1–2 focused sessions across the four phases.

## Open Risks & Assumptions

- **SDK API drift.** Names/signatures verified at `@anthropic-ai/claude-agent-sdk` v0.3.195 (`tool`, `createSdkMcpServer`, `query`, `mcpServers` object map, `mcp__code-review__review`); the implementer should confirm against the installed version.
- **Forcing one tool call** relies on `tools: []` + `allowedTools` + system prompt (no `tool_choice` in the Agent SDK); the no-call case is handled as an error.
- **Lint coupling.** If `packages/**` isn't excluded from the root ESLint program, `eslint .` and the Lefthook pre-commit hook will fail on the new package.
- **`tsc` deprecation** under TS 6 may need `--ignoreDeprecations 6.0` (per project memory).

## Success Criteria (Summary)

- A reviewer can pipe a real `git diff` and get back a Zod-valid, readable review generated through the `review` tool call.
- `--json` and `--model` work; missing key / empty stdin fail clearly with non-zero exit.
- One root `pnpm install`; the Astro app's build and lint stay green.
